# pdf2md-pro Fase 0+1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pipeline PDF→Markdown nativa (pymupdf4llm) con sidecar di provenienza, estrazione immagini, CLI, corpus di test sintetico e harness di valutazione.

**Architecture:** Pacchetto `src/pdf2md_pro` con engine intercambiabili dietro `PageResult`; pipeline orchestratrice che assembla Markdown + provenienza; eval separato in `eval/`.

**Tech Stack:** Python 3.11+, pymupdf4llm, PyMuPDF (fitz), pytest, argparse (stdlib).

## Global Constraints

- CLI solo stdlib (argparse), nessuna dipendenza extra oltre pymupdf4llm/pymupdf/pytest.
- Output mai sovrascritto senza `--force`.
- Ogni blocco MD tracciato a pagina nel sidecar `<nome>.provenance.json`.
- Commit format: `<type>: <description>` (feat/fix/test/docs/chore), niente co-author.
